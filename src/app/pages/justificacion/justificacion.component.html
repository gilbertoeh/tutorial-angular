Justificación

Angular, es la opción framework de desarrollo front-end de código abierto mantenida por Google, para agilizar el proceso de creación de aplicaciones web de una sola página.

1) ¿Para qué sirve Angular?

Angular utiliza un enfoque basado en componentes para estructurar y organizar aplicaciones en piezas reutilizables, con el fin de facilitar la gestión de proyectos complejos. Su propósito principal es simplificar el desarrollo y mantenimiento de aplicaciones de una sola página (Single Page Applications, SPA), donde la mayor parte del contenido se carga dinámicamente sin necesidad de recargar toda la página web.

Herramientas y funcionalidades integradas que ofrece Angular:

  • Two-way Data Binding: permite sincronizar automáticamente los datos entre el modelo y la vista, eliminando la necesidad de escribir lógica adicional para actualizar ambos.
  • Inyección de Dependencias: permite una gestión eficiente de servicios y módulos, mejorando la modularidad y reutilización del código.
  • Directivas: facilitan la manipulación del DOM de manera declarativa.
  • Router Integrado: proporciona una navegación fluida entre diferentes vistas o componentes.
  • Testing Integrado: incluye herramientas para pruebas unitarias y de integración, lo que mejora la calidad del software.

2) ¿Qué aplicaciones tiene Angular?

Angular tiene un amplio rango de aplicaciones, desde soluciones simples hasta sistemas empresariales complejos. Algunos ejemplos destacados incluyen:

  • Desarrollo Web de SPA: Ideal para aplicaciones de una sola página que requieren una experiencia fluida y rápida, como paneles de administración, plataformas de aprendizaje en línea o sistemas de comercio electrónico.
  • Aplicaciones Empresariales: Angular se utiliza en entornos corporativos para crear aplicaciones robustas con altos requerimientos de seguridad y escalabilidad.
  • Aplicaciones Progresivas (PWA): Permite desarrollar aplicaciones web que funcionan como aplicaciones nativas, con soporte para el trabajo sin conexión.
  • Sistemas de Gestión de Contenidos (CMS): Gracias a su flexibilidad y facilidad de integración con APIs, es una opción popular para crear CMS personalizados.
  • Aplicaciones en Tiempo Real: Su capacidad para interactuar con WebSockets lo hace adecuado para sistemas que requieren actualizaciones en tiempo real, como chats o aplicaciones financieras.

3) ¿Sobre qué lenguajes se apoya Angular?

Angular utiliza varios lenguajes clave:

  • TypeScript: Es el lenguaje principal para escribir la lógica de las aplicaciones en Angular.
  • HTML: Sirve para definir la estructura de las vistas.
  • CSS (o preprocesadores como SCSS y LESS): Se utiliza para diseñar y estilizar los componentes y vistas.
  • JavaScript: Aunque Angular utiliza TypeScript, este se compila a JavaScript para ejecutarse en los navegadores.

4) ¿Qué prerrequisitos necesita Angular?

Para comenzar con Angular, se requiere cumplir con ciertos prerrequisitos técnicos:

  • Conocimientos básicos de JavaScript: Comprender conceptos modernos de ECMAScript (ES6+), como let y const, funciones flecha, desestructuración, módulos y promesas.
  • La instalación de Node.js y npm: Node.js permite ejecutar scripts de JavaScript en el entorno del servidor.
  • Familiaridad con HTML y CSS: Conocimiento sobre cómo estructurar páginas web y aplicar estilos básicos.
  • Conocer el CLI de Angular: La interfaz de línea de comandos facilita tareas comunes de pruebas y compilación.
