Instalación

La instalación de Angular implica preparar un entorno de desarrollo adecuado para garantizar un rendimiento óptimo y una configuración coherente.

1) ¿Se precisa instalación?

Sí, Angular requiere una instalación previa de herramientas clave para su correcto funcionamiento. Aunque en realidad no necesita una instalación directa, ya que es accesible a través del CLI (Command Line Interface), el entorno de desarrollo debe configurarse con ciertos componentes:

  • Node.js y npm: Angular utiliza Node.js como entorno de ejecución para gestionar dependencias y ejecutar scripts durante el desarrollo.
  • Angular CLI: Una herramienta que simplifica la creación, configuración y mantenimiento de proyectos Angular.
  • Editor de código (opcional): Un IDE como Visual Studio Code mejora significativamente la productividad.

2) ¿Qué pasos hay que realizar para realizar la instalación?

  • 2.1) Instalar Node.js: Descarga e instala la versión LTS desde www.nodejs.org.
  • 2.2) Verificar la instalación: Comprueba que Node.js y npm estén correctamente instalados ejecutando los comandos:
    • node -v # Muestra la versión de Node.js
    • npm -v # Muestra la versión de npm
  • 2.3) Instalar Angular CLI: Ejecuta el comando: npm install -g @angular/cli@latest.
  • 2.4) Verificar instalación: Usa el comando ng version para confirmar.
  • 2.5) Crear un nuevo proyecto: Usa el comando ng new nombre-del-proyecto. Se te solicitarán configuraciones iniciales.

3) ¿Versiones necesarias?

  • Node.js: Al menos v16.x o superior. Verifica compatibilidad en la documentación oficial.
  • npm: Las versiones recientes (v8 o superior) son compatibles.
  • Angular CLI: Usa la última versión con npm install -g @angular/cli@latest.
  • Framework Angular: El CLI instalará la última versión automáticamente o una específica con ng new nombre-del-proyecto --version=15.2.0.

4) ¿Rutas para tener en cuenta?

  • Directorio global del CLI:
    • Windows: C:\Users\<usuario>\AppData\Roaming\npm
    • Mac/Linux: /usr/local/lib/node_modules
  • Directorio del proyecto: Trabaja desde la carpeta del proyecto para ejecutar comandos ng.
  • Ruta al binario ng: Si el comando ng no se reconoce, verifica que la carpeta esté incluida en el PATH del sistema.
