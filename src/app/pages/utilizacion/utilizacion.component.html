Utilización

1) ¿Cómo empezar a crear una aplicación base?

Crear una aplicación base en Angular implica utilizar el Angular CLI, una herramienta que automatiza tareas comunes como la configuración inicial del proyecto, la generación de componentes y la gestión de dependencias. Este proceso puede dividirse en los siguientes pasos técnicos:

1.1) Instalación del Angular CLI

Antes de comenzar, verifica que el CLI de Angular esté instalado globalmente en tu sistema. Si no es así, instálalo ejecutando: 

npm install -g @angular/cli

Este comando descarga e instala el CLI en el entorno global, permitiendo su acceso desde cualquier directorio del sistema. Confirma la instalación con: 

ng version

Esto muestra información sobre las versiones instaladas del CLI, Angular, Node.js y npm.

1.2) Creación del proyecto

Utiliza el comando ng new para crear una nueva aplicación Angular. Este comando configura automáticamente la estructura del proyecto, instala las dependencias y genera los archivos base. Por ejemplo: 

ng new mi-aplicacion

Durante la ejecución del comando, se solicitan varias opciones:

  • Habilitar enrutamiento: Responde "Sí" o "No" dependiendo de si deseas configurar el sistema de rutas en este momento.
  • Estilo predeterminado: Selecciona entre opciones como CSS, SCSS, SASS o LESS. Este será el preprocesador o lenguaje de estilos utilizado en tu proyecto.

El comando genera una carpeta con el nombre del proyecto (mi-aplicacion) que incluye una estructura predefinida con archivos como:

  • src/: Carpeta principal del proyecto.
  • app/: Contiene el módulo raíz (app.module.ts) y el componente principal (app.component.*).
  • assets/: Carpeta para almacenar recursos estáticos como imágenes o fuentes.
  • environments/: Archivos de configuración para diferentes entornos (producción, desarrollo, etc.).
  • angular.json: Archivo de configuración del CLI.
  • package.json: Define las dependencias del proyecto.
  • tsconfig.json: Configuración de TypeScript.

1.3) Configuración inicial

Después de la creación del proyecto, navega al directorio generado: 

cd mi-aplicacion

El proyecto ya está listo para ejecutarse, pero es posible que desees realizar personalizaciones iniciales, como ajustar configuraciones en angular.json o modificar la plantilla en app.component.html.

2) ¿Cómo visualizar los resultados de la ejecución?

Angular CLI incluye un servidor de desarrollo integrado que permite compilar y visualizar la aplicación en tiempo real. Este servidor es útil para pruebas locales y para observar cómo se reflejan los cambios en la interfaz.

2.1) Iniciar el servidor de desarrollo

Ejecuta el siguiente comando para compilar y ejecutar la aplicación: 

ng serve

Esto desencadena una serie de procesos técnicos:

  • Compilación con Webpack: Angular utiliza Webpack internamente para compilar los archivos TypeScript, HTML y CSS en un paquete JavaScript que puede ser interpretado por el navegador.
  • Servidor local: Angular CLI inicia un servidor HTTP en el puerto predeterminado 4200. El servidor admite recarga automática (live reload), lo que significa que cualquier cambio en los archivos del proyecto se compila automáticamente y se refleja en el navegador.

2.2) Acceso a la aplicación

Abre un navegador y accede a: 

http://localhost:4200

El navegador mostrará la página predeterminada de Angular, que incluye un mensaje de bienvenida generado automáticamente por el CLI.

2.3) Personalización de la vista principal

El contenido inicial mostrado en el navegador proviene del archivo app.component.html. Puedes personalizar esta vista modificando dicho archivo: 

<h1>Hola Mundo desde mi aplicación Angular</h1>

Guarda los cambios y verifica que se reflejen automáticamente en el navegador, gracias a la funcionalidad de live reload.

2.4) Personalización adicional

Puedes modificar otros archivos para ajustar aspectos clave de la aplicación base:

  • Estilos: Modifica src/styles.css para aplicar estilos globales o utiliza los archivos CSS de cada componente para estilos específicos.
  • Variables de entorno: Configura valores específicos en los archivos de src/environments/ según el entorno de desarrollo o producción.
  • Configuración de puertos: Si deseas utilizar un puerto diferente, ejecuta: 
    ng serve --port 4300
