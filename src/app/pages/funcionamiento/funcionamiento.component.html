Funcionamiento

Angular actúa sobre estructuras modulares como páginas, secciones y elementos del DOM, gestionados a través de componentes y módulos. Los ficheros del framework se modifican siguiendo una estructura organizada y utilizando herramientas como Angular CLI y editores compatibles. Además, Angular puede interactuar con recursos externos y aplicaciones mediante APIs, librerías o componentes reutilizables, extendiendo su alcance más allá del entorno inmediato.

1) ¿Actúa sobre páginas, ficheros, secciones…?

  • 1.1) Single Page Applications (SPA):

    Angular está diseñado principalmente para aplicaciones de una sola página, en las que el contenido se actualiza dinámicamente sin recargar toda la página. Esto se logra mediante un sistema de enrutamiento que controla la visualización de diferentes vistas desde un solo archivo HTML base (index.html).

  • 1.2) Estructura Modular:
    • 1.2.1) Módulos: Angular organiza la funcionalidad en módulos (@NgModule), que agrupan componentes, servicios y directivas relacionados.
    • 1.2.2) Componentes: Cada componente es responsable de una sección específica de la interfaz de usuario (UI). Actúan sobre vistas definidas en plantillas HTML (component.html) y están vinculados a archivos de estilos (component.css) y lógica (component.ts).
  • 1.3) DOM (Document Object Model): Angular interactúa directamente con el DOM mediante sus directivas (*ngIf, *ngFor, etc.) para manipular elementos visuales de manera declarativa, basada en datos dinámicos.
  • 1.4) Ficheros de Configuración: Angular utiliza archivos como angular.json, tsconfig.json, y package.json para configurar aspectos técnicos como la compilación, el uso de TypeScript y las dependencias del proyecto.

2) ¿Cómo se editan/modifican los ficheros que actúan en el framework?

  • 2.1) Componentes:
    • Cada componente incluye tres archivos principales: HTML, CSS/SCSS y TypeScript.
    • Para cambiar la lógica, se edita el archivo .ts, añadiendo métodos, propiedades o importando dependencias necesarias.
    • La vista se ajusta editando el archivo .html con elementos HTML estándar y directivas de Angular.
    • Los estilos se personalizan en el archivo .css o .scss.
  • 2.2) Módulos: Los módulos (module.ts) gestionan la inclusión de componentes, servicios y otras funcionalidades.
  • 2.3) Directivas y Servicios:
    • Directivas: Modifican elementos del DOM. Se configuran en @NgModule y se usan directamente en las plantillas HTML.
    • Servicios: Contienen lógica compartida y se inyectan en los componentes donde son necesarios.
  • 2.4) Configuración: Archivos clave como angular.json o tsconfig.json permiten ajustar configuraciones globales.
