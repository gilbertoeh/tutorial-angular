Primeros Pasos

El siguiente ejercicio proporciona una introducción a Angular. Te mostrará cómo los componentes interactúan dentro de la arquitectura del framework. Aunque simple, este "Hola Mundo" sienta las bases para comprender conceptos más avanzados como inyección de dependencias, directivas y servicios, esenciales para el desarrollo de aplicaciones robustas.

1) Estructura inicial de Angular

Cuando creas un proyecto Angular con el comando ng new, se genera una estructura estándar que incluye:

  • Módulos (app.module.ts), que define los componentes, servicios y dependencias que forman parte de la aplicación.
  • Componentes (app.component.*), que manejan la lógica y el diseño de cada parte de la interfaz de usuario.
  • Archivos de configuración (angular.json, package.json), que controlan las dependencias y configuraciones del proyecto.
  • Sistema de enrutamiento (opcional), que permite navegar entre vistas si habilitado durante la creación del proyecto.

2) Creación del proyecto Angular

2.1) Para crear un nuevo proyecto Angular, ejecuta el siguiente comando: 

ng new hola-mundo-app

Nota: Durante la ejecución, se te pedirá confirmar opciones como:

  • Routing: Puedes elegir "No" para este ejemplo simple.
  • Estilos: Selecciona CSS como lenguaje para los estilos.

2.2) Accede al proyecto navegando al directorio generado: 

cd hola-mundo-app

3) Creación del componente "Hola Mundo"

3.1) Genera un componente personalizado usando Angular CLI: 

ng generate component hola-mundo

Este comando genera automáticamente cuatro archivos en la carpeta src/app/hola-mundo/:

  • hola-mundo.component.ts: Contiene la lógica del componente.
  • hola-mundo.component.html: Define la vista (plantilla).
  • hola-mundo.component.css: Aplica estilos específicos al componente.
  • hola-mundo.component.spec.ts: Archivos de pruebas unitarias para el componente.
